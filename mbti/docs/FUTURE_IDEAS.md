# 심화 로드맵 (FUTURE_IDEAS)

지금까지의 PersonaForge는 **"페르소나 하나를 잘 만들고 검증하는"** 입구가 정교한 반면,
**"여러 페르소나로 무엇을 하는가"** 하는 출구가 빈약하다. Sim Francisco의 본질이 사실
*개인이 아니라 집단(1만 명)을 시뮬레이션해 집계로 예측*하는 것이었음을 떠올리면, 우리가
버린 "다중 에이전트 + 집계" 부분에 가장 큰 미개발 잠재력이 있다.

아래는 가치 순으로 정리한 심화 항목. 체크박스로 진행 상황을 관리한다.

---

## 1순위 — 페르소나 간 상호작용 (arena: 대화)
- [ ] 여러 페르소나를 한 자리에 두고 **서로 대화**시키기 (레제 ↔ 마키마, INTJ ↔ ENFP)
- [ ] 각자 자기 페르소나로 상대 발언에 반응 (ChatEngine 두 개 이상을 번갈아 구동)
- [ ] 벡터 거리(`ValueVector.distance`)로 두 페르소나의 **충돌/궁합 예측**
- [ ] 라운드 수·발화 순서·중재자(사회자) 옵션
- 의의: 단순 1:1 챗봇을 넘어 Sim Francisco의 "다중 에이전트" 정신 복원

## 2순위 — 집계/예측 복원 (arena: 폴링)
- [ ] 여러 페르소나에게 **같은 질문**을 던지고 답을 모으기 (Sim Francisco poll 복원)
- [ ] 예: 16개 MBTI 유형에게 같은 딜레마 → 유형별로 답이 어떻게 갈리는지 집계
- [ ] 답을 축/유형/살리언스별로 **집계·시각화**(텍스트 차트 등)
- [ ] "이 질문에 어떤 유형이 찬성/반대 경향"을 데이터로 제시
- 의의: Sim Francisco의 polling → aggregation 을 캐릭터/MBTI 도메인에 진짜 복원

## 3순위 — 페르소나 검증 루프 (blind identification)  ← **지금 구현**
- [x] LLM에게 페르소나 답변만 보여주고 "무슨 유형/캐릭터 같냐"를 **역추정**하게 함
- [x] 맞히면 페르소나가 잘 만들어진 것 → 처음으로 **품질을 객관적으로 측정**
- [x] 적대적 비평가의 자연스러운 확장; Sim Francisco backtest 정신
- [x] MBTI: 답변 → 유형 4글자 역추정 정확도 / 캐릭터: 후보 중 식별
- 의의: "INTJ 페르소나가 진짜 INTJ다운가"를 잴 방법이 처음으로 생김

## 4순위 — 페르소나 블렌딩
- [ ] 벡터 산술로 페르소나 혼합: "레제 70% + 마키마 30%"
- [ ] 단일 축 조정 변형: "INTJ인데 따뜻함만 높인" 버전
- [ ] 블렌딩 결과의 prose 재생성 + provenance에 혼합 비율 기록
- 의의: 캐릭터 창작 도구로서의 재미; 벡터 표현의 자연스러운 활용

---

## 같이 고려할 것 — 덜어내기 (가지치기)
입구만 비대하고 출구가 빈약한 불균형을 바로잡기 위해, 실사용 후 아래를 평가:
- [ ] `sufficiency.py`(276줄) — 5차원이 다 필요한가, 1~2개로 충분한가
- [ ] `critic.py`(273줄) — 4종 검사가 실제로 가치를 주는가
- [ ] rubric 키워드 채점 — LLM 채점과 너무 다르면 키워드 경로 폐기 고려
- 기준: `docs/TESTING_ON_MACBOOK.md` 의 "실행 후 회고" 결과를 근거로
