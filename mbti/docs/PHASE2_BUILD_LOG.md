## 내가 자율로 정한 것 (+이유)

**D1. 스타일 디렉티브 언어 = 영어.**
이유: 기존 코드(`build_mbti_prose`, `FUNCTIONS`)가 전부 영어라 일관성·유지보수↑. 출력 언어는
`build_assistant_system_prompt(language=...)`로 분리 — 모델이 영어 지시 읽고 지정 언어로 답함
(PoC에서 검증된 패턴). 한국어 제품이어도 무방.

**D2. AssistantEngine은 ChatEngine 재사용 안 하고 별도 신규.**
이유: ChatEngine의 적대적 비평가(persona_break 등)는 **롤플레이 전용**이라 어시스턴트엔 해로움
— 예: "나는 AI야"가 어시스턴트에선 *필수*인데 persona_break로 거부됨. 그래서 가볍고 분리된
엔진. Phase 1 코드는 한 줄도 안 건드림(mode 분기 = 별 진입점).

**D3. P2.1 도구 = mock 스텁만(weather/news), 실제 연동 X.**
이유: 로드맵상 P2.1=스타일 코어, P2.2=도구. 자는 중 도구 설계(네이티브 vs 자체) 세부 결정을
못 받으니, 인터페이스(`Tool` 프로토콜)만 깔고 실연동은 P2.2로. 어시스턴트는 도구 없이도 일반
지식 Q&A는 다 됨.

**D4. 가드레일 = 주로 프롬프트(BLOCK C) + 경량 위기감지 유틸.**
이유: 반아첨·능력우선은 프롬프트로 제어(PoC에서 효과 확인). 위기감지는 키워드(EN+KO) 기반
탐지 후 안전 지시를 추가 주입하는 경량 후크. 깊은 강제는 평가(eval)로 모니터.
